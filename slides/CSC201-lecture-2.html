2’s complement representation

  • Same as signed-bit except the MSB of an $n$-bit number has weight $-2^{n-1}$
  • The 2’s complement of an n-digit binary number $x$ is defined as $2^n - x$ if $x \neq 0$ and 0 otherwise.
  • Overcomes the problems of sign bit

Review

Sign-bit vs 2’s complement representations

Similarities:

  • No difference for positive numbers
  • For negative numbers, MSB = 1 in both representations

Difference:

  • The value of an n-bit negative number in sign-bit repr. is the $-1 \times$ (the decimal value of the remaining n-1 bits)
  • The value of an n-bit negative number in 2’s complement repr. is $-2^{n-1} +$ (the decimal value of the remaining n-1 bits)

How does it work?

Say we want to subtract $n$ from $m$, both of which are $p$-bit numbers

  • Find $m$ + $r$'s complement of $n$, i.e. $m + (2^p - n)$
  • If $m > n$, the sum will overflow and the carry bit will be discarded.
  • If $m < n$, the sum wont overflow and is equal to 2’s complement of $(n-m)$ i.e. $2^p - (n - m)$
